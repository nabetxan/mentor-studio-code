## 2. Phase 2.1: Setup ダイアログ未表示バグ

### 症状

Setup コマンド（`Mentor Studio: Setup Mentor`）を実行した結果：
- ファイル生成は全て成功（docs/mentor/rules/ 配下、データファイル、.mentor-studio.json）
- CLAUDE.md 作成確認ダイアログが表示されなかった
- Setup 完了後の Reload Window ダイアログも表示されなかった
- CLAUDE.md は物理的に削除済み（git 上は未コミットの D 状態）、CLAUDEBackup.md が存在

### 原因仮説

- `showInformationMessage` の `await` が正しく動作していない、または前段の処理でエラーが握りつぶされている可能性
- VSCode の通知設定による抑制の可能性

### 修正方針

#### Step 1: 調査用ログ追加

Setup コマンド内の主要ポイントにコンソールログを追加し、再現テストで実行フローを確認する。

```typescript
console.log("Mentor Studio Setup: starting CLAUDE.md handling");
// ... CLAUDE.md logic
console.log("Mentor Studio Setup: CLAUDE.md handling complete");
console.log("Mentor Studio Setup: showing reload dialog");
// ... reload dialog
console.log("Mentor Studio Setup: complete");
```

#### Step 2: CLAUDE.md 作成ロジック変更（バグとは独立したデザイン改善）

確認ダイアログの必要性を見直す。これはバグ修正ではなく、UX改善として独立して行う。

**変更前:**
```
存在しない → ダイアログ「Create CLAUDE.md?」→ Yes なら作成
存在する → 内容に mentorRef がなければダイアログ「Append?」
```

**変更後:**
```
存在しない → 自動作成（ダイアログなし）
存在する & mentorRef なし → ダイアログ「Append?」
存在する & mentorRef あり → スキップ
```

理由: Setup を実行した時点で Mentor Studio を使う意思があるため、新規 CLAUDE.md の作成に確認は不要。既存ファイルへの追記のみ確認する。

#### Step 3: Reload ダイアログの調査・修正

Step 1 のログで Reload ダイアログ（extension.ts:139-145）が到達しているか確認する。到達しているのに表示されない場合は、`showInformationMessage` の呼び出し方を検証する。

#### Step 4: Setup 完了通知の改善

ファイル生成結果のサマリーを Output Channel に出力する。Output Channel はモジュールレベルで1回だけ作成し、再利用する。

```typescript
// モジュールレベル（extension.ts のトップ）
let outputChannel: vscode.OutputChannel | undefined;

function getOutputChannel(): vscode.OutputChannel {
  if (!outputChannel) {
    outputChannel = vscode.window.createOutputChannel("Mentor Studio");
  }
  return outputChannel;
}

// Setup コマンド内
const ch = getOutputChannel();
ch.appendLine("=== Setup Results ===");
ch.appendLine(`Created: ${createdFiles.join(", ") || "none (all existed)"}`);
ch.appendLine(`Skipped: ${skippedFiles.join(", ") || "none"}`);
ch.appendLine(`CLAUDE.md: ${claudeAction}`);
ch.show(true); // true = preserveFocus
```

これにより、ダイアログが表示されなくても結果を確認できる。

### Codicon 導入

#### パッケージインストール

`@vscode/codicons` を `extension/package.json` の `dependencies` に追加（devDependencies ではない。フォントファイルを実行時に参照するため）。

```bash
cd extension && npm install @vscode/codicons
```

#### localResourceRoots の更新

`sidebarProvider.ts` の `resolveWebviewView` で `localResourceRoots` に Codicon のパスを追加する。

```typescript
localResourceRoots: [
  vscode.Uri.joinPath(this.extensionUri, "webview", "dist"),
  vscode.Uri.joinPath(this.extensionUri, "node_modules", "@vscode", "codicons", "dist"),
],
```

#### HTML で Codicon CSS を読み込む

`sidebarProvider.ts` の `getHtml` メソッドで URI を生成し、`<link>` を追加する。

```typescript
const codiconsUri = webview.asWebviewUri(
  vscode.Uri.joinPath(
    this.extensionUri,
    "node_modules",
    "@vscode",
    "codicons",
    "dist",
    "codicon.css",
  ),
);
```

```html
<link href="${codiconsUri}" rel="stylesheet" />
```

#### CSP の更新

現在の CSP:
```
default-src 'none'; style-src ${webview.cspSource} 'unsafe-inline'; script-src 'nonce-${nonce}';
```

変更後（`font-src` を追加）:
```
default-src 'none'; style-src ${webview.cspSource} 'unsafe-inline'; script-src 'nonce-${nonce}'; font-src ${webview.cspSource};
```

#### VSIX パッケージング

`@vscode/codicons` は `dependencies` にあるため、`vsce package` 実行時に `node_modules/@vscode/codicons/` が自動的に含まれる。追加のコピー手順は不要。

### 旧 CSS の削除

Actions コンポーネントのリファクタに伴い、以下の旧 CSS クラスを `main.css` から削除する：

- `.snippet-card` — `.snippet-btn` に置き換え
- `.snippet-copy` — ボタン全体がクリッカブルになったため不要
- `.snippet-list` の `list-style: none` — `<ul>` → `<div>` に変更するため不要。`.snippet-list` 自体は維持するが、`list-style` ルールは削除
- `.snippet-title` — 維持（ボタン内で引き続き使用）

## 5. 設計判断の記録

| 判断 | 選択 | 理由 |
|---|---|---|
| CLAUDE.md 新規作成 | 確認なしで自動作成 | Setup 実行 = 使う意思がある。ダイアログ不要 |
| Setup 結果通知 | Output Channel に出力 | ダイアログが出なくても結果確認可能 |
| Actions UI | カード全体をボタン化 | 押しやすさ向上。クリック領域拡大 |
| コピーアイコン | Codicon `copy` → `check` | VSCode ネイティブ。卒業帽テーマに近いスタイル |
| 説明文配置 | h3 直下 | 最初に目に入る位置。控えめなスタイルで邪魔にならない |
| パスガイド | 未設定時のみ表示 | 設定済みの場合は不要。初回のみ案内 |
