# Phase 2.4: サイズ変更 / Zoom Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add zoom in/out controls to the Settings tab, allowing the user to scale the entire webview UI up or down.

**Architecture:** A CSS custom property `--zoom` on `<body>` controls the base font-size via `calc()`. Zoom level is stored in `.mentor-studio.json` as an integer percentage (e.g. `100`, `125`). Zoom +/- buttons in Settings post a `setZoom` message. Extension persists the value to config. Webview applies the zoom on load from config and on change.

**Tech Stack:** TypeScript, React, plain CSS, VSCode Extension API, Vitest

### Task 2: Add zoom-driven CSS

**Files:**
- Modify: `extension/webview/src/styles/main.css`

- [ ] **Step 1: Update body styles**

Replace the existing `body` rule:

```css
body {
  margin: 0;
  padding: 0;
  font-family: var(--vscode-font);
  font-size: calc(var(--vscode-font-size, 13px) * var(--zoom, 1));
  color: var(--vscode-foreground);
  background: var(--vscode-sideBar-background);
}
```

This way, setting `--zoom: 1.25` on the body element scales all `em`-relative or `inherit` sizes. Elements with explicit `px` sizes will need to be converted or kept as-is for structural elements (borders, gaps, etc.) — only font-sizes and paddings that should scale need to use `em` or `inherit`.

Key insight: since most text elements use `font-size: inherit` or relative values, and the base `body` font-size changes, most of the UI will scale automatically. Only hardcoded `px` font-sizes (like `11px`, `12px`, `20px`) need attention.

- [ ] **Step 2: Convert hardcoded font-sizes to relative units**

Update `.stat-label`, `.stat-value`, `.actions-description`, `.snippet-copied-text`, `.setting-label`, `.setting-btn`, `.setting-path`, `.setting-warning`, `.setting-guide`, `.status`, `.gap-detail` to use `em`-based sizes relative to body:

```css
/* Examples of conversions (base = 13px at zoom 1.0): */
.stat-label { font-size: 0.846em; }     /* was 11px → 11/13 */
.stat-value { font-size: 1.538em; }     /* was 20px → 20/13 */
.actions-description { font-size: 0.923em; } /* was 12px → 12/13 */
.snippet-copied-text { font-size: 0.846em; } /* was 11px */
.setting-label { font-size: 0.846em; }

```

Note: Keep structural dimensions (border, border-radius, progress-bar height) in `px` — they shouldn't scale with text.

Add zoom controls section (below language toggle):

```tsx
const MIN_ZOOM = 70;
const MAX_ZOOM = 150;
const ZOOM_STEP = 10;

// Inside Settings return, after language toggle setting-item:
<div className="setting-item">
  <div className="setting-label">{t("settings.zoom", locale)}</div>
  <div className="zoom-controls">
    <button
      className="zoom-btn"
      onClick={() => { if (zoom > MIN_ZOOM) onZoomChange(zoom - ZOOM_STEP); }}
      disabled={zoom <= MIN_ZOOM}
    >
      −
    </button>
    <span className="zoom-value">{zoom}%</span>
    <button
      className="zoom-btn"
      onClick={() => { if (zoom < MAX_ZOOM) onZoomChange(zoom + ZOOM_STEP); }}
      disabled={zoom >= MAX_ZOOM}
    >
      +
    </button>
    {zoom !== 100 && (
      <button
        className="zoom-btn zoom-btn-reset"
        onClick={() => onZoomChange(100)}
      >
        {t("settings.zoomReset", locale)}
      </button>
    )}
  </div>
</div>
```

Add `updateZoom` method (same pattern as `updateLocale`):
```typescript
private async updateZoom(zoom: number): Promise<void> {
  const wsRoot = vscode.workspace.workspaceFolders?.[0]?.uri;
  if (!wsRoot) {
    return;
  }
  const configUri = vscode.Uri.joinPath(wsRoot, ".mentor-studio.json");
  try {
    const raw = await vscode.workspace.fs.readFile(configUri);
    const config = JSON.parse(
      Buffer.from(raw).toString(),
    ) as MentorStudioConfig;
    config.zoom = zoom;
    await vscode.workspace.fs.writeFile(
      configUri,
      Buffer.from(JSON.stringify(config, null, 2) + "\n"),
    );
    this.latestConfig = config;
    this.postMessage({ type: "config", data: config });
  } catch (err) {
    console.error("Failed to update zoom", err);
  }
}
```
