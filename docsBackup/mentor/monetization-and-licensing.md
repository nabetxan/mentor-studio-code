2026年03月23日

## 3つの方向性

### 1. フルOSS（広く使ってもらう路線）

**ライセンス：** MIT or Apache 2.0

| メリット | デメリット |
|---|---|
| 採用障壁が最低。誰でもすぐ使える | 収益ゼロ |
| コントリビューション・PRが来やすい | 企業がフォークして商用化しても止められない |
| ポートフォリオ・実績として最強 | サーバー費用は自腹 |
| Exercism（200万ユーザー）もこのモデル | |

**向いてるケース：** 「まず使ってくれる人を増やしたい」「就活・転職のポートフォリオにしたい」「コミュニティを作りたい」

---

### 2. オープンコア（OSS + 有料クラウド）

**ライセンス：** Extension本体はMIT、APIサーバー部分はAGPLv3 or BSL

```
無料（OSS）                    有料（月額）
─────────────────────         ─────────────────────
VSCode Extension               クラウド同期（API/DB）
ローカルダッシュボード            複数プロジェクト横断分析
File Watcher                   Webダッシュボード
ローカルJSON完結                 学習レポート生成
                               チーム/教室向け機能
```

**価格感（競合ベース）：**
- 個人: $5-8/月（Khanmigo $4、Execute Program $39を参考に、間をとる）
- チーム/教室: $10-15/人/月

**向いてるケース：** 「無料で広めつつ、ヘビーユーザーからは収益を得たい」。Phase 1がまさにOSS部分、Phase 2が有料部分、と今の設計にぴったりハマる。

---

### 3. スポンサーウェア / GitHub Sponsors

**ライセンス：** FSL (Functional Source License) or 独自ライセンス

- GitHub Sponsorsで月額目標を設定
- 目標達成 → MITとして公開
- 達成前 → スポンサーのみアクセス可能

Calmly（CSSフレームワーク）やSidekiq（Ruby）がこのモデルで成功している。

**向いてるケース：** 「個人開発を持続可能にしたいが、最終的にはOSSにしたい」

## マネタイズアイデア集

「まず広める」を前提に、収益化の手段を幅広く洗い出す。組み合わせOK。

### A. フリーミアム（使用量制限）

| 制限軸 | 無料 | 有料 |
|---|---|---|
| プロジェクト数 | 2プロジェクトまで | 無制限 |
| 質問履歴の保存期間 | 直近3ヶ月 | 無制限 |
| クラウド同期 | なし（ローカルのみ） | あり |
| エクスポート | JSON のみ | CSV / PDF レポート |

**ポイント：** 学習初期は1プロジェクトで十分なので、無料で始めやすい。2つ目のプロジェクトも無料にすることで「便利さ」を体感させてから課金に誘導。

---

### B. プレミアム機能（まだない機能を有料に）

#### B-1. 学習アナリティクス強化
- **週次/月次レポート自動生成** — 「今週の正答率推移」「先月との比較」をMarkdownやPDFで出力
- **学習ヒートマップ** — GitHubのcontribution graphのような「今日どれだけ学んだか」の可視化
- **トピック別の成長曲線グラフ** — 「TypeScriptは3週間前から正答率が20%→70%に」
- **AIによる学習アドバイス** — 弱点パターンから「次に勉強すべきこと」を提案

#### B-2. 復習・クイズ強化
- **スペースドリピティション** — 間違えた問題を最適なタイミングで自動再出題（Anki的）
- **カスタムクイズ生成** — トピックを指定して、過去の間違いベースでクイズを自動生成
- **デイリーチャレンジ** — 毎日1問、弱点から出題（通知つき）

#### B-3. チーム / 教室向け
- **教師ダッシュボード** — 生徒全員の進捗・弱点を一覧で確認
- **クラス管理** — 課題の割り当て、提出状況の追跡
- **生徒間の匿名ランキング** — モチベーション施策

#### B-4. 連携 / エクスポート
- **Notion / Obsidian エクスポート** — 学習ログを好みのツールに出力
- **GitHub Discussions連携** — 間違えた概念をGitHub Discussionsに自動投稿して質問
- **Webダッシュボード** — ブラウザで詳細分析（Phase 3相当）

---

### C. 広告モデル

| 方式 | 具体例 | 現実度 |
|---|---|---|
| **Webダッシュボードにバナー広告** | Google AdSense等。サイドバーExtension内は広告禁止（VSCode規約） | Phase 3以降なら可能 |
| **学習レポートページにスポンサー枠** | 「このレポートは○○の提供でお届け」 | Webダッシュボード実装後 |
| **おすすめ教材のアフィリエイト** | 弱点トピックに関連するUdemy/書籍をリンク。「TypeScriptが苦手なあなたへ」 | Extension内でも可能 |

**注意：** VSCode Extension内に広告を表示するのはマーケットプレイスの規約的にグレー〜NG。やるならWebダッシュボード側で。

### D. スポンサーシップ

| 方式 | 具体例 |
|---|---|
| **GitHub Sponsors** | 個人スポンサーを募集。Tierごとに特典（ロゴ掲載、優先サポート、限定機能Early Access） |
| **企業スポンサー** | プログラミングスクール・教材会社に「Powered by ○○」枠を販売 |
| **Open Collective** | OSSプロジェクトとして透明な資金管理。支出もオープンに |
| **READMEスポンサー枠** | 人気OSSがよくやる方式。GitHubのREADMEにスポンサーロゴを表示 |

---

### E. コンテンツ / コミュニティ課金

| 方式 | 具体例 |
|---|---|
| **プレミアムプロンプトテンプレート** | 「React Hook完全理解コース」「TypeScript型パズル50問」等のメンタープロンプトセットを販売 |
| **学習コミュニティ（Discord有料チャンネル）** | 月額制で、同じツールを使う学習者同士の交流 + 質問サポート |
| **ブログ / YouTube** | 開発過程やAIメンタリングのノウハウをコンテンツ化。直接収益 + ツールの認知拡大 |

---

### F. データ活用（匿名・オプトイン前提）

| 方式 | 具体例 |
|---|---|
| **匿名学習データの集計レポート** | 「全ユーザーの苦手トピックランキング」「初学者がつまずきやすい概念TOP10」を公開→認知拡大 |
| **教材会社へのインサイト提供** | 「TypeScriptのどの概念が最も正答率が低いか」を匿名統計として教材開発に活用（要オプトイン） |

**注意：** プライバシーには最大限配慮。学習データは個人情報に近いので、匿名化 + 明示的オプトインが必須。

## おすすめの組み合わせ（「広める」前提）

```
Phase 1（今）
  └─ 完全無料 / MIT ライセンス
  └─ GitHub Sponsors 開設（支援したい人向け）
  └─ 開発過程をブログ / SNSで発信 → 認知拡大

Phase 2（ユーザーがつき始めたら）
  └─ ローカル機能は無料のまま
  └─ クラウド同期 + Webダッシュボード = 有料（$5/月程度）
  └─ 3プロジェクト以上 = 有料
  └─ アフィリエイト（弱点に応じた教材おすすめ）

Phase 3（軌道に乗ったら）
  └─ チーム / 教室プラン
  └─ プレミアム分析（成長曲線、AIアドバイス）
  └─ 企業スポンサー枠
  └─ 匿名統計レポート公開
```

## 現段階での現実的な判断

**フルOSSを推す理由：**
- ターゲットが「Claude Code Pro + VSCodeで学習してる人」というかなりニッチな層
- ニッチな市場で有料にすると、そもそもユーザーが集まらないリスク
- 最初のユーザー100人を獲得することが一番大事なフェーズ
- OSSで実績を作ることは、開発者としてのキャリアにもプラス

**オープンコアを推す理由：**
- Phase 1（ローカル完結）= 無料、Phase 2（API/DB）= 有料、と設計が自然に分かれている
- サーバー費用（Vercel / Supabase）は無料枠を超えるとコストがかかる
- 「クラウド同期」は明確な価値があるので課金しやすい

## ライセンス比較

| ライセンス | 特徴 | 代表例 |
|---|---|---|
| **MIT** | 最も自由。誰でもコピー・改変・商用利用OK | React, VS Code |
| **Apache 2.0** | MITに似ているが特許保護あり | Kubernetes, Goose |
| **AGPLv3** | フォークしても公開義務あり。SaaS化を防げる | Grafana, MongoDB(旧) |
| **BSL** | 一定期間後にOSSに変わる。商用利用に制限 | Sentry, MariaDB |

**おすすめの組み合わせ：**
- シンプルに行くなら → **MIT一本**
- クラウド部分を守りたいなら → Extension: MIT + API: AGPLv3
